# 标签页功能说明

## 功能概述

在主框架上添加了标签页（Tab）功能，用于保存和管理用户打开过的页面，方便用户在不同页面之间快速切换。

## 功能特性

### 1. 基本功能
- ✅ 自动记录用户访问的页面
- ✅ 点击标签可快速切换到对应页面
- ✅ 支持关闭单个标签
- ✅ 首页（仪表盘）标签固定，不能关闭

### 2. 右键菜单功能
在任意标签上右键点击，可以看到以下选项：
- **刷新页面**：重新加载当前页面内容
- **关闭当前**：关闭当前标签（固定标签除外）
- **关闭其他**：关闭除当前标签和固定标签外的所有标签
- **关闭左侧**：关闭当前标签左侧的所有标签（固定标签除外）
- **关闭右侧**：关闭当前标签右侧的所有标签
- **关闭所有**：关闭所有标签（固定标签除外）

### 3. 视觉设计
- 渐变色高亮显示当前激活的标签
- 鼠标悬停时显示关闭按钮
- 平滑的过渡动画效果
- 横向滚动支持（标签过多时）

## 技术实现

### 文件结构

```
src/
├── store/
│   └── modules/
│       └── tabs.js                    # 标签页状态管理
├── layout/
│   ├── index.vue                      # 主布局（已集成标签页组件）
│   └── components/
│       └── TabsView.vue               # 标签页组件
└── views/
    └── redirect/
        └── index.vue                  # 重定向页面（用于刷新功能）
```

### 核心实现

1. **标签页状态管理** (`src/store/modules/tabs.js`)
   - 使用 Pinia 管理标签页状态
   - 提供标签的增删改查功能
   - 支持页面缓存管理（keep-alive）

2. **标签页组件** (`src/layout/components/TabsView.vue`)
   - 显示所有打开的标签
   - 处理标签点击、关闭等交互
   - 实现右键菜单功能

3. **路由集成**
   - 在 `src/router/index.js` 中配置固定标签（affix: true）
   - 在 `src/store/modules/permission.js` 中同步固定标签配置
   - 创建重定向页面用于页面刷新

## 如何添加固定标签

如果需要将某个页面设置为固定标签（不能关闭），只需在路由配置的 meta 中添加 `affix: true`：

```javascript
{
  path: "your-path",
  name: "YourPage",
  component: () => import("@/views/your-page.vue"),
  meta: {
    title: "页面标题",
    icon: "your-icon",
    affix: true,  // 设置为固定标签
  },
}
```

## 注意事项

1. 标签页会自动记录所有有 `name` 和 `meta.title` 的路由
2. 如果不希望某个页面被缓存，可以在路由 meta 中设置 `noCache: true`
3. 隐藏的路由（meta.hidden: true）不会显示在标签页中
4. 刷新页面时，标签页历史会清空，只保留当前页面标签

## 浏览器兼容性

支持所有现代浏览器：
- Chrome/Edge (最新版本)
- Firefox (最新版本)
- Safari (最新版本)
